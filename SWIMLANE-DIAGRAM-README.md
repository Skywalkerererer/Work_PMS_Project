## üìê Diagram Structure

### Swimlanes (Top to Bottom)

| # | Lane | Color | Key Activities |
|---|------|-------|----------------|
| 1 | **Tenant / Occupant** | Light Blue | Receive notices, make payments, raise disputes |
| 2 | **PMS - Billing & Dunning Engine** | Purple | Automated scheduler, fee application, case management |
| 3 | **Communications Service** | Orange | Multi-channel delivery (Email/SMS/Letter) |
| 4 | **AR / Collections Team** | Green | Manual review, payment plans, dispute handling |
| 5 | **Property Manager** | Pink | Investigate disputes, approve adjustments |
| 6 | **Finance / Legal Escalation** | Red | Final notices, legal referrals |

### Shape Legend

- **Rectangles (rounded)**: Process tasks/activities
- **Diamonds**: Decision points (Yes/No branches)
- **Ellipse**: Start event (daily scheduler)
- **Solid arrows**: Process flow
- **Dashed arrows**: Information flow (e.g., payment notifications)

### Color Coding

- **Blue**: System/automated processes
- **Green**: Successful outcomes (payments, resolutions)
- **Yellow**: Decision points (require evaluation)
- **Orange**: Communication actions
- **Red**: Escalation and legal actions
- **Pink**: Manual investigation/approval

## üîÑ Process Flow Summary

### Main Process Path (Happy Path)
```
Daily Scheduler ‚Üí Identify Overdue ‚Üí Exclusion Check ‚Üí Compute Level 
‚Üí Apply Late Fee (if needed) ‚Üí Create Case ‚Üí Send Reminder 
‚Üí Tenant Pays ‚Üí Auto-Allocate ‚Üí Log & Close
```

### Dispute Path
```
Tenant Receives Notice ‚Üí Raises Dispute ‚Üí AR Flags Dispute 
‚Üí Property Manager Investigates ‚Üí Decision (Valid/Invalid) 
‚Üí Adjust or Reject ‚Üí Log & Resume/Close
```

### Escalation Path
```
No Payment ‚Üí Collections Review ‚Üí Age/Amount Check 
‚Üí Final Notice ‚Üí Legal Threshold Check ‚Üí DCA/Legal Referral 
‚Üí Log & External Collection
```

## üéØ Key Decision Points in the Diagram

| Decision | Location | Criteria | Outcomes |
|----------|----------|----------|----------|
| **Account Excluded?** | PMS Lane | Dispute, payment plan, manual hold | Yes ‚Üí Skip / No ‚Üí Continue |
| **Fee Threshold Crossed?** | PMS Lane | D+14 or configured date | Yes ‚Üí Apply fee / No ‚Üí Skip to case |
| **Delivery Failure?** | Comms Lane | Email/SMS bounce | Yes ‚Üí Call task / No ‚Üí Continue |
| **Payment Received?** | PMS Lane | Payment posted | Yes ‚Üí Allocate / No ‚Üí Escalate |
| **Partial Payment?** | PMS Lane | Amount < invoice total | Yes ‚Üí Plan offer / No ‚Üí Escalate |
| **Dispute Valid?** | PM Lane | Investigation outcome | Yes ‚Üí Credit note / No ‚Üí Resume |
| **Age > 45d OR Amt > Threshold?** | Legal Lane | Escalation criteria | Yes ‚Üí Final notice / No ‚Üí Continue |
| **Age > 60d OR Amt > HKD X?** | Legal Lane | Legal referral criteria | Yes ‚Üí DCA/Legal / No ‚Üí Log |
